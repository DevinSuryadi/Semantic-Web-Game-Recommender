Saya sedang mengerjakan proyek tugas akhir mata kuliah Semantic Web.

Judul proyek:
GameFeel: Sistem Rekomendasi Game Berdasarkan Kemiripan Semantik Menggunakan RDF, Ontology, dan SPARQL.

Deskripsi proyek:
Saya ingin membangun sistem rekomendasi game yang tidak hanya berdasarkan genre, rating, popularitas, atau platform, tetapi berdasarkan kemiripan semantik atau “feel” dari sebuah game. Sistem ini akan membantu pengguna menemukan game lain yang memiliki nuansa permainan, suasana, tema, mekanik gameplay, mode bermain, subgenre, gaya combat, perspektif kamera, tingkat kesulitan, pacing, platform, dan art style yang mirip dengan game yang dipilih.

Contoh:
Jika pengguna menyukai Elden Ring, sistem tidak hanya mencari game bergenre Action RPG atau Open World, tetapi juga mencari game dengan karakteristik seperti dark fantasy, challenging combat, exploration, boss fight, mysterious world, high difficulty, dan atmospheric experience.

Komponen utama proyek:
1. Mengambil data game dari RAWG API.
2. Menyimpan data mentah dan data hasil cleaning ke staging lokal menggunakan Prisma.
3. Membangun ontology untuk domain rekomendasi game.
4. Mengonversi data game ke RDF/Turtle.
5. Mengunggah RDF ke Apache Jena Fuseki sebagai SPARQL Endpoint.
6. Membuat query SPARQL untuk pencarian dan rekomendasi game.
7. Membuat backend API untuk menghubungkan frontend dengan Fuseki dan data staging jika diperlukan.
8. Membuat frontend React untuk pencarian game, detail game, rekomendasi game serupa, similarity score, dan alasan rekomendasi.

Catatan penting:
- Saya tidak ingin menggunakan Streamlit.
- Saya ingin menggunakan React + TypeScript.
- Saya ingin menggunakan pnpm.
- Saya ingin menggunakan Prisma.

- Prisma tidak menggantikan RDF, ontology, atau Fuseki.
- Fuseki tetap digunakan sebagai SPARQL Endpoint utama.
- Saya tidak menggunakan Supabase.
- Saat deploy, data utama untuk rekomendasi tetap diambil dari Jena Fuseki.
- Fokus deployment adalah aplikasi yang membaca RDF/SPARQL dari Fuseki, bukan sinkronisasi ke database cloud tambahan.
- Fokus utama proyek tetap Semantic Web, RDF, Ontology, SPARQL, Knowledge Graph, dan explainable recommendation.
- Sistem tidak menggunakan machine learning kompleks.

Definisi feel:
- Feel game direpresentasikan sebagai gabungan beberapa dimensi, bukan satu label tunggal.
- Mood menggambarkan atmosfer emosional.
- Theme menggambarkan konteks naratif atau setting.
- GameplayMechanic menggambarkan aksi inti pemain.
- GameMode menggambarkan pola interaksi pemain.
- SubGenre menggambarkan label genre-spesifik seperti FPS, MOBA, atau Roguelike.
- CombatStyle menggambarkan pola tempur seperti Turn-Based atau Real-Time.
- Perspective menggambarkan sudut pandang bermain seperti First-Person atau Isometric.
- QualityTier hanya dipakai sebagai metadata tambahan bila diperlukan.

Aturan kamus:
- Setiap game boleh memiliki lebih dari satu nilai untuk Mood, Theme, GameplayMechanic, GameMode, SubGenre, CombatStyle, Perspective, dan Tag.
- Difficulty, Pacing, ArtStyle, dan QualityTier dibatasi pada nilai yang paling relevan.
- Jika suatu istilah tidak cocok dengan kamus final, istilah itu tetap disimpan sebagai Tag, bukan dipaksa masuk ke kelas semantik lain.
- Semua nilai semantik direpresentasikan sebagai individu/instance pada class masing-masing, bukan free text.
- Label seperti Positive, Very Positive, Mixed, atau Similar tidak dipakai jika hanya memakai RAWG. Jika ingin penilaian kualitas, gunakan QualityTier yang diturunkan dari rating/metacritic.

Aturan pemetaan RAWG ke atribut semantik:
- `genres` dipetakan langsung ke class Genre dan, bila relevan, ke SubGenre.
- `platforms` dipetakan langsung ke class Platform.
- `developers` dipetakan langsung ke class Developer.
- `publishers` dipetakan langsung ke class Publisher.
- `tags` adalah sumber utama untuk Mood, Theme, GameplayMechanic, GameMode, SubGenre, CombatStyle, Perspective, Difficulty, Pacing, ArtStyle, QualityTier, dan Tag.
- `description` menjadi sumber pendukung untuk inferensi berbasis kata kunci jika tags terlalu umum atau tidak lengkap.
- `rating`, `metacritic`, dan `playtime` tetap menjadi data properties serta ranking metadata, bukan label semantik.
- Jika dibutuhkan kategori kualitas, turunkan dari rating/metacritic menjadi QualityTier.
- `background_image` dipetakan ke imageUrl.
- `id`, `slug`, `released`, dan `name/title` dipakai sebagai identitas dan metadata game.

Metode rekomendasi:
Sistem mencari kesamaan atribut semantik antar game menggunakan RDF dan SPARQL. Similarity score dihitung berdasarkan kesamaan genre, subgenre, mood, theme, gameplay mechanic, game mode, combat style, perspective, difficulty, pacing, art style, platform, tag, dan quality tier jika diperlukan.

Bobot similarity score:
- Genre sama: +3
- SubGenre sama: +3
- Mood sama: +2
- Theme sama: +2
- Gameplay mechanic sama: +2
- GameMode sama: +2
- CombatStyle sama: +2
- Difficulty sama: +1
- Pacing sama: +1
- Perspective sama: +1
- Art style sama: +1
- Platform sama: +1
- Tag sama: +1
- QualityTier sama: +1 (opsional, hanya sebagai tiebreaker)

Contoh hasil rekomendasi:
Input: Elden Ring

Rekomendasi:
1. Dark Souls III
   Alasan: sama-sama memiliki genre Action RPG, mood Dark, mekanik Boss Fight, tingkat kesulitan Hard, dan dikembangkan oleh FromSoftware.

2. Bloodborne
   Alasan: sama-sama memiliki suasana Dark, gameplay menantang, mekanik Boss Fight, dan tema Gothic Horror.

3. The Witcher 3
   Alasan: sama-sama memiliki genre RPG, dunia eksploratif, tema Fantasy, dan elemen Open World.
